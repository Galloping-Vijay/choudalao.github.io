&lt;p&gt;&amp;nbsp;由于博客升级,要把原来的系统和数据全都替换掉,安装完之后发现原来的数据表全都还在,根据表名前缀的区别,可以批量的删掉原来的数据表,操作如下:&lt;/p&gt;&lt;p&gt;执行sql语句:&lt;/p&gt;&lt;pre lay-lang=&quot;JavaScript&quot;&gt;&lt;code class=&quot;JavaScript&quot;&gt;Select CONCAT( 'drop table ', table_name, ';' ) 
FROM information_schema.tables 
Where table_name LIKE 'pre_%';&lt;/code&gt;&lt;/pre&gt;&lt;p&gt;&lt;span&gt;解释:like`pre_%`,其中pre_为你需要删除的表前缀,根据你自己的需求修改,&lt;/span&gt;&lt;/p&gt;&lt;p&gt;执行查询,会自动把匹配的表生成出 drop table table_name这样的SQL语句,&lt;/p&gt;&lt;p&gt;运行结果如下:&lt;/p&gt;&lt;p&gt;&lt;img src=&quot;http://www.yuemeet.com/data/upload/2017-10-27/59f2a343ab8ba.png&quot; title=&quot;59f2a343ab8ba.png&quot; alt=&quot;59f2a343ab8ba.png&quot; _src=&quot;/data/upload/2017-10-27/59f2a343ab8ba.png&quot;&gt;&lt;/p&gt;&lt;p&gt;批量复制一下到记事本或者et之类的编辑工具中,确定你的sql语句是否正确.这么一来也可以安全的审核一下语句,避免误操作.&lt;/p&gt;&lt;p&gt;最好用记事本打开,这样可以去掉样式,检查之后复制到sqlyog里面运行:&lt;/p&gt;&lt;p&gt;&lt;img src=&quot;http://www.yuemeet.com/data/upload/2017-10-27/59f2a7a5621d7.png&quot; title=&quot;59f2a7a5621d7.png&quot; alt=&quot;59f2a7a5621d7.png&quot; _src=&quot;/data/upload/2017-10-27/59f2a7a5621d7.png&quot;&gt;&lt;/p&gt;&lt;p&gt;ok,成功删除相同表前缀的文章了!&lt;/p&gt;&lt;h3&gt;拓展:&lt;br&gt;&lt;/h3&gt;&lt;p&gt;批量修改表前缀:&lt;/p&gt;&lt;pre lay-lang=&quot;JavaScript&quot;&gt;&lt;code class=&quot;JavaScript&quot;&gt;Select CONCAT( 'ALTER TABLE ', table_name, 'RENAME TO ', table_name,';' ) 
FROM information_schema.tables 
Where table_name LIKE 'uc_%';\
&lt;/code&gt;&lt;/pre&gt;&lt;p&gt;&lt;br&gt;&lt;/p&gt;&lt;pre class=&quot;brush:sql;toolbar:false;&quot;&gt;批量复制一下到记事本或者&amp;nbsp;et之类的编辑工具中,然后批量替换&amp;nbsp;RENAME&amp;nbsp;TO&amp;nbsp;uc&amp;nbsp;成&amp;nbsp;RENAME&amp;nbsp;TO&amp;nbsp;你想要的表前缀完成后&amp;nbsp;再执行&lt;br&gt;&lt;/pre&gt;&lt;p&gt;&lt;br&gt;&lt;/p&gt;&lt;p&gt;&lt;br&gt;&lt;/p&gt;
